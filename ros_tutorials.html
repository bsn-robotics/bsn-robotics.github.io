http://wiki.ros.org/ROS/Tutorials/InstallingandConfiguringROSEnvironment

Вы должны выполнять эту команду в каждой новой оболочке терминала, для того чтобы были доступны команды ROS, если вы не добавили эту команду в .bashrc. Это команды позволяет использовать несколько разных дистербьютивов ROS на одном компьютере.

source /opt/ros/indigo/setup.bash

You will need to run this command on every new shell you open to have access to the ROS commands, unless you add this line to your .bashrc. This process allows you to install several ROS distributions (e.g. indigo and kinetic) on the same computer and switch between them.


Создать рабочее пространство ----------


Из директории с рабочим пространством catkin_ws нужно выполнить комнанду:

source devel/setup.bash
Для проверки: 
$ echo $ROS_PACKAGE_PATH
/home/youruser/catkin_ws/src:/opt/ros/kinetic/share
Чтобы не вводить каждый раз можно добавить в  .bashrc:
source  /home/sergius/catkin_test/devel/setup.sh

Урок №7

rosservice args /service - Отображает аргументы сервиса
∙ rosservice call /service - Вызывает сервис с представленными аргументами
∙ rosservice find msg-type - Ищет сервисы по их типу
∙ rosservice info /service - Выводит информацию о сервисе
∙ rosservice list - Список активных сервисов
∙ rosservice type /service - Отображает тип сервиса
∙ rosservice uri /service - Выводит ROSPRC URI сервиса

rosservice call clear

rosservice call /spawn 2 2 0.2 ""

rosparam set            set parameter
rosparam get            get parameter
rosparam load           load parameters from file
rosparam dump           dump parameters to file
rosparam delete         delete parameter
rosparam list           list parameter names
